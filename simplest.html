筆記

1. 如果從學校電腦抓12G的檔案 , leo 套件要自己下載 ,安裝到 site- package 裡面。


2. launchLeo.py 和 start 可以從老師提供的壓縮檔中找到 , 複製到相同路徑即可 ( start 多個啟動程序 )。


3. 檢查分支 : git bramch

新建分支 : git branch (分支名稱)

切換分支 : git checkout (分支名稱)


4. 因所在的分支不同 , 資料夾所顯示的檔案也有所不同


5. 新建的分支會複製原有的分支檔案

6. dir : 檢察所屬分支的檔案數量和狀況


7. git config (--global) user.name .... , 括弧內若省略 , 此次僅為單 一1次 , 若沒有省略 , 則以後皆為默認


8. REM = REMARK , .bat 檔 = batch 批次檔 , -s = set 設定


9. reveal.leo 存檔之後所產生的simplest.html檔案 , 可以很方便的檢查近端leo的編輯狀況


10. github 產生的倉儲可以依造自己所需 , 產生 https 或者 ssh 的clone URL


11. leo 檔案不正當的關閉 , 將導致 leo editor 出現錯誤而無法正常開啟

12. 如果leo出現問題無法開啟 , 試著將 home 底下的 .leo 資料夾刪除

13. 如果推回遠端有問題 , 可能和 .git 檔 、 路徑錯誤 或 者其他問題

14. 無法正常出現分頁頁面( io ) 網址的原因 : github 尚未進行 信箱認證 、 抓錯檔案等原因


15. 如果將老師的檔案推到自己倉儲的 master , 遠端頁面顯示404 File not found , 但是推到 gh-pages 卻會顯示網頁 , 代表 github 的 gh-pages 能支援靜態網站


16. 如果分支有新增資料，要先展開、提交之後才能再切換分支


17. git rm 檔案名稱.副檔名 ( 移除檔案在近端動作即可 )

18. git rm 資料夾名稱 -r 可刪除資料夾( 含內部文件 )


19. git push origin :分支名稱 ( 刪除github 分支 )
